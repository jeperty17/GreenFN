# REVIEW Folder

This folder contains per-task review files for completed setup tasks.

## Current Task Reviews

- `REACT.md` — React + Vite setup summary, reproduce commands, route checks, and file-type purposes.
- `SHADCN_TAILWIND.md` — ShadCN + Tailwind setup summary, reproduce commands, verification steps, and file-type purposes.
- `NODE_EXPRESS.md` — Node + Express setup summary, middleware/module-boundary checks, write-endpoint validation checks, and file-type purposes.
- `POSTGRES_SUPABASE.md` — Supabase/PostgreSQL setup verification, secure env handling checks, and file-type purposes.
- `AUTH_APPROACH.md` — Authentication decision record (chosen approach, rationale, migration path, and validation checks).
- `RLS_STRATEGY.md` — Row-level security strategy setup (policy script, execution, and verification checks).
- `PRISMA_SCHEMA.md` — Prisma schema model/relationship baseline verification and migration-state checks.
- `PRISMA_WORKFLOW.md` — Prisma client generation + migration workflow script verification.
- `PRISMA_SEEDING.md` — Baseline local pipeline/test-data seeding setup and idempotency checks.
- `PRISMA_ERROR_MAPPING.md` — Prisma-to-HTTP error mapping strategy and middleware verification.
- `AI_PROVIDER_SELECTION.md` — AI provider/model decision for MVP summarization and drafting.
- `AI_SECRET_HANDLING.md` — Backend-only AI API key/secret handling setup verification.
- `AI_SERVICE_WRAPPER.md` — AI wrapper setup for templates, retries, and token/cost controls.
- `AI_LOGGING_POLICY.md` — AI logging policy enforcement for metadata-only, non-raw-text logging.
- `VERCEL_PROJECT.md` — Vercel frontend project creation and local link verification.
- `RAILWAY_SERVICE.md` — Railway backend project/service creation and linkage verification.
- `DEPLOYMENT_ENV_VARS.md` — Production env variable setup across Railway backend and Vercel frontend.
- `CORS_ALLOWLIST.md` — Production CORS allowlist configuration and verification.
- `CICD_SETUP.md` — GitHub Actions CI/CD for main-branch deploys and PR preview deployments.

## Usage

1. Open the relevant task review file.
2. Run commands in order to reproduce and validate behavior.
3. Use route checks and file checks to confirm expected setup output.
